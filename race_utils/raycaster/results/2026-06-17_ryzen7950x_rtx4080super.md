## 2) Real-time feasibility — budget 25 ms/cycle @ 40 Hz

SIM = 2 lidars × 2200 beams (176 k rays/s). PF = 4000 particles × 100 beams (16 M rays/s).
headroom = 25 ms / measured.

| method | SIM ms | SIM ↑ | PF ms | PF ↑ | verdict |
|---|--:|--:|--:|--:|---|
| numba f1tenth_gym | 0.213 | 117× | **35.34** | 0.7× | **PF FAIL** |
| range_libc rm | 0.092 | 272× | 20.42 | 1.2× | PF tight |
| range_libc cddt | 0.084 | 299× | 8.64 | 2.9× | PF OK |
| range_libc pcddt | 0.072 | 349× | 8.05 | 3.1× | PF OK |
| range_libc glt | 0.024 | 1048× | **1.56** | **16×** | PF OK (best CPU) |
| jax DT-march (GPU) | 1.009 | 25× | 2.78 | 9× | PF OK (needs GPU) |
| jax DT-march (CPU) | 0.618 | 40× | **58.34** | 0.4× | **PF FAIL** |

→ SIM: all pass huge. PF: numba & jax-CPU fail even here; glt(16×) safest for low-end; cddt/pcddt(3×) OK desktop / risky laptop; jax-GPU(9×) only where an NVIDIA GPU exists.

## 4) Precomputed LUT (backend='lut') — `f` map, θ_disc 112, max 10 m

| step | result |
|---|---|
| build oracle (CDDT) + materialize 40.3M entries | ≈ 1.9 s (one-time) |
| save `.npz` (compressed) | 6.9 MB |
| load | 158 ms |
| PF batch 4000×100 (pure numpy, no range_libc) | 2.1 ms → 11.7× real-time |
| accuracy vs pcddt | 1.4 cm (112-bin θ quantization) |

`RaycastEngine` unified-library self-test (same pose, f map):

| backend | scan mean | PF 4000×100 | vs pcddt |
|---|--:|--:|--:|
| pcddt (range_libc) | 8.90 m | 4.44 ms (5.6×) | 0.0 cm |
| cddt (range_libc) | 8.88 m | 4.45 ms (5.6×) | 1.9 cm |
| **lut (numpy, portable)** | 8.89 m | **2.14 ms (11.7×)** | 1.4 cm |

→ LUT backend is correct, fastest for the PF batch, and needs no range_libc at query
time (copy the 6.9 MB `.npz` to a NUC/Mac mini and it runs).

## Caveats this run
- `raycast_test/range_libc` (deploy branch) is **Python 2** → not py3-buildable; used CAC `slam/range_libc` (py3, = the build the particle_filter uses).
- `range_libc rmgpu` (CUDA) **skipped**: no system CUDA toolkit + CUDA 13 vs old kernels.
- jax raycaster is a **faithful reimplementation** of the `jax_pf` distance-transform sphere-tracing (github install blocked); same algorithm.
